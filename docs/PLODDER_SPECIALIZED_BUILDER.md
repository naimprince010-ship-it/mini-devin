# Plodder — The Specialized Builder (রোডম্যাপ)

**স্ট্যাক ফোকাস:** Next.js · Supabase · Tailwind CSS · Drizzle ORM  

**প্রডাক্ট ভিশন:** ইউজার এক বাক্যে চাহিদা বললে (যেমন মেডিসিন শপ ইনভেন্টরি ড্যাশবোর্ড) Plodder একই স্ট্যাকে ধারাবাহিকভাবে দেবে: ডাটা মডেল (Drizzle স্কিমা + মাইগ্রেশন নীতি), Supabase-সেফ অ্যাক্সেস প্যাটার্ন, এবং Tailwind/shadcn-ভিত্তিক UI—লিন্ট/বিল্ড/মিনিমাল টেস্ট দিয়ে যাচাই করা।

## ১) সাকসেস ক্রাইটেরিয়া (প্রথম ~৩ মাস, রিয়ালিস্টিক)

| মেট্রিক | টার্গেট |
|--------|---------|
| স্ট্যাক অ্যালাইনমেন্ট | নতুন টাস্কের বেশিরভাগ ক্ষেত্রে জেনেরেটেড কোড একই ফোল্ডার/কনভেনশন অনুসরণ করে |
| স্কিমা কোয়ালিটি | প্রতিটি ফিচারে টেবিল, ইনডেক্স, FK, টাইমস্ট্যাম্প/সফট ডিলিট (প্রয়োজনে) নথিভুক্ত |
| UI | লোডিং / এরর / এম্পটি স্টেট + এক টোকেন সিস্টেম (Tailwind) |
| ভেরিফিকেশন | `npm run build` পাস; Drizzle চেক স্ক্রিপ্ট থাকলে সেটাও পাস |
| টোকেন/কস্ট | স্ট্যাক প্রিসেট + প্লেবুক ব্যবহারে টাস্ক প্রতি রিপিটেড কনটেক্সট কমানো (ট্র্যাক করা) |

## ২) স্ট্র্যাটেজি: এজেন্ট জেনেরিক, স্ট্যাক অপিনিয়নেটেড

- **এজেন্ট কোর** (টার্মিনাল, এডিটর, ভেরিফাই, গিট/গিটহাব) জেনেরিকই রাখা—রিমেইনটেইনেবল।
- **স্পেশালাইজেশন** আসবে: অফিসিয়াল স্টার্টার টেমপ্লেট + প্লেবুক + UI প্রিসেট + ভেরিফিকেশন প্রোফাইল দিয়ে।

## ৩) ফেজড রোডম্যাপ

### ফেজ ০ — বেসলাইন (১–২ সপ্তাহ)

**লক্ষ্য:** একই স্ট্যাকে সবাই একই কনভেনশন অনুসরণ করে।

- **স্টার্টার টেমপ্লেট (রিপো):** Next (App Router) + Tailwind + Drizzle + `@supabase/supabase-js`, `.env.example`, ফোল্ডার কনভেনশন (`db/`, `app/`, `components/`).
- **ডক এক পাতায়:** `PLAYBOOK.md` বা `PLODDER_STACK.md` — স্কিমা নিয়ম, RLS কখন লাগবে, সার্ভার অ্যাকশন vs ক্লায়েন্ট, কমিটে সিক্রেট নয়।
- **সেশন প্রিসেট:** `stack: next-supabase-drizzle` (বা সমতুল্য) বাছাই করলে সিস্টেম প্রম্পটে স্ট্যাক ব্লক ইনজেক্ট।

**আউটপুট:** নতুন সেশন খালি ফোল্ডার না থেকে টেমপ্লেট ক্লোন দিয়ে শুরু করার পথ।

### ফেজ ১ — বিল্ডার ওয়ার্কফ্লো (২–৪ সপ্তাহ)

**লক্ষ্য:** ন্যাচারাল ল্যাঙ্গুয়েজ → স্ট্রাকচার্ড প্ল্যান → কোড।

- টাস্ক ইনটেকে ছোট স্ট্রাকচার: এন্টিটি, রোল, CRUD স্ক্রিন, রিপোর্ট/ফিল্টার—LLM-কে গাইড করা।
- **ডাটা মডেল প্রথম:** Drizzle স্কিমা → কুয়েরি/সার্ভার অ্যাকশন → UI।
- **UI কিট:** shadcn/Radix + Lucide; ড্যাশবোর্ড = টেবিল + ফিল্টার + সাইড শিট ইত্যাদি প্যাটার্ন প্লেবুকে।

**আউটপুট:** একই ধরনের অ্যাপ বারবার একই ফাইল লেআউটে পড়া যায়।

### ফেজ ২ — ভেরিফিকেশন ও নিরাপত্তা (২–৩ সপ্তাহ)

**লক্ষ্য:** ভুল স্কিমা বা ভাঙা বিল্ড প্রোডে না যাওয়া।

- `package.json` স্ক্রিপ্ট: `db:generate`, `db:check` (বা `drizzle-kit check`), `build`, `lint`।
- অটো ভেরিফিকেশন: Node বিল্ড ব্লকিং; Drizzle চেক অপশনাল ব্লকিং (এনভ ফ্ল্যাগ)।
- সুপাবেস: সার্ভার কী vs anon—প্লেবুকে স্পষ্ট।

**আউটপুট:** “টাস্ক কমপ্লিট” এর আগে টুল আউটপুটে বিল্ড পাস দেখানো।

### ফেজ ৩ — প্রোডাক্ট ও ডক্স (চলমান)

- টেমপ্লেট আপডেট ও মাইগ্রেশন নিয়ম ডক।
- গোল্ডেন উদাহরণ টাস্ক (রিগ্রেশন): ইনভেন্টরি, সিম্পল CRM, অর্ডার বোর্ড।
